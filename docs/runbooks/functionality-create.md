# Functionality: Create

In order to create a new functionality in the workspace, you need to go through the following steps:

- Determine the functionality's type: 'abstract', 'visual', 'visual+', 'mixed', 'mixed+'.

**Tip!** [Click here](../getting-started/library-types-and-their-relationship.md#functionality-types) to read more about different types.

- Create PRD & TFS files of the functionality.

Based on the functionality type, generate & build the required libs:

- Generate & build a '_map_' lib.
- Generate & build a '_data-access_' lib.
- Generate & build a '_ui_' lib.
- Generate & build a '_feature_' lib.
- Generate & build a '_page_' lib.

**Important!** It's important to keep all of the created/updated libs' `README.md` files up-to-date according to the finalized codes. The Definition of Done (DoD) for a lib is to have a working ready-to-use copy-paste documentation! We basically need to be able to copy-paste the lib's sample codes in its `README.md` file, and start using/testing the lib, and be able to see the final results without any errors.
